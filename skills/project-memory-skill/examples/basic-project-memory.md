# Basic Project Memory Example

## Starting A Task

1. Read `.ai/AGENTS.md` for permanent project rules.
2. Read `.ai/DECISIONS.md` before changing architecture or workflow.
3. Read `.ai/MEMORY.md` for prior findings.
4. Read `.ai/HANDOFF.md` for current state.
5. Read `.ai/FAILED_ATTEMPTS.md` before retrying a failed approach.

## During Work

- Add durable discoveries to `.ai/MEMORY.md`.
- Add important choices to `.ai/DECISIONS.md`.
- Add failed approaches to `.ai/FAILED_ATTEMPTS.md`.
- Keep `.ai/HANDOFF.md` current when the task is not finished.

## Ending A Task

Update `.ai/HANDOFF.md` with:

- Current goal
- Completed work
- Remaining blockers
- Next steps
- Modified files
